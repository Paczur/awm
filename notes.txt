global:
- configuration split to different files
- modules have to be testable
- keep things as small and simple as possible
- prefer clean restart over being dynamic
- restore EVERYTHING on restart

shortcuts:
- modifiers: shift ctrl alt
- 100 shortcut max
- auto-repeat
- normal mode only
- press only
- every shortcut declared by:
  * modifiers
  * keysym
  * function pointer
- functions:
  * init from array
  * handle
- shortcut array size:
  * 100 shortcuts * 1 byte = 100 bytes - search array
  * 100 shortcuts * (4/8) byte = 50 bytes - check array
  * 100 shortcuts * 8 bytes = 800 bytes - function array
  * Summed = 950 bytes
  * access is only 100 + 1 + 8 = 109 bytes AT WORST

layout:
- tiling only, for now
- 10 workspaces max
- dynamic binding of workspaces together
- positional navigation
- minimization
- swapping
- destroying
- restoring
- grid like layout (no trees)
- resizing
- 4 windows per workspace
- robust error handling
- good way to restore state
- workspaces change name to window name present
- all windows with border unless only one visible
- fullscreen mode
- think what to do with multi-monitor

bar:
- builtin so no communication overhead
- runs bash scripts in separate thread
- displays:
  * current mode
  * workspaces
  * minimize stack
  * blocks (script output)
- embedded launcher

x:
- dumb wrapper for all xcb functions
